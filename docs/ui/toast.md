# Toast

## Назначение

Toast — единая система уведомлений проекта **Olesya Martin Designer**.

Цель решения:
- централизованная настройка уведомлений
- единый стиль и поведение
- отсутствие дублирования ToastContainer

---

## Расположение в проекте

```
shared/ui/Toast
```

Компонент является обёрткой над библиотекой `react-toastify`.

---

## Принцип использования

Компонент `<Toast />` подключается **один раз** на уровне страницы или layout
(например, AdminPanel или App).

```tsx
<Toast />
```

## Типы уведомлений

В проекте используются следующие типы Toast-уведомлений:

- `toast.success` — успешное завершение пользовательского действия  
  (данные сохранены, операция выполнена, действие подтверждено)

- `toast.error` — ошибка выполнения операции  
  (ошибка API, серверная ошибка, критический сбой)

- `toast.info` — информационное уведомление  
  (процесс начался, статусное сообщение, нейтральная информация)

Другие типы (`warning`, `dark` и т.п.) не используются без отдельного решения.

---

## Когда использовать Toast

Toast используется для:
- результата пользовательского действия
- ошибок API
- важных информационных сообщений

---

## Когда НЕ использовать Toast

❌ для валидации форм  
❌ для постоянных подсказок  
❌ как замену UI-состояний (loading / empty / error)

---

## Ограничения и запреты

❌ Не создавать собственные `ToastContainer`  
❌ Не подключать `react-toastify` напрямую в компонентах  
❌ Не менять глобальные настройки без фиксации решения
