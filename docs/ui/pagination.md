# Pagination

## Назначение

Pagination — единый компонент пагинации проекта **Olesya Martin Designer**.

Компонент создан для того, чтобы:
- не дублировать логику пагинации
- обеспечить единый UX во всех местах проекта
- избежать прямого использования сторонней библиотеки в фичах

Использование альтернативных реализаций пагинации **не допускается** без обсуждения.

---

## Расположение в проекте

```
shared/ui/Pagination
```

Компонент является обёрткой над библиотекой `react-paginate`.

---

## Принцип использования

Pagination используется как **управляемый компонент**.  
Состояние текущей страницы хранится в родительском компоненте.

Обязательные пропсы:
- `totalItems` — общее количество элементов
- `itemsPerPage` — количество элементов на странице
- `selectedPage` — текущая страница (0-based)
- `onPageChange` — обработчик смены страницы

Пример:

```tsx
<Pagination
  totalItems={items.length}
  itemsPerPage={1}
  selectedPage={activeIndex}
  onPageChange={setActiveIndex}
/>
```

---

## Что можно настраивать

- внешний вид через `Pagination.module.scss`
- количество элементов на странице
- логику переключения страниц (через родительский компонент)

---

## Ограничения и запреты

❌ Не использовать `react-paginate` напрямую  
❌ Не копировать логику пагинации в другие компоненты  
❌ Не создавать кастомные pagination без согласования
